# 毕业论文检查 Skill

针对 **杭州电子科技大学信息工程学院本科毕业设计/论文** 的 Word 格式检查与内容质量评估。

整套流程基于 `docx` skill 的 `unpack.py` 解包后，用 Python 标准库 `xml.etree` 直接解析原始 OOXML，**无需任何第三方依赖**。

---

## 操作步骤

### 第 1 步：解包 .docx

找到 `docx` skill 的安装路径（通常是 `.claude/skills/docx/scripts/office/unpack.py`），然后：

```bash
python3 <docx-skill路径>/scripts/office/unpack.py "<论文.docx路径>" /tmp/thesis_unpacked/
```

### 第 2 步：运行检查

```bash
python3 scripts/check_format_xml.py /tmp/thesis_unpacked/
```

脚本输出 JSON，包含 `critical` / `warning` / `passed` / `content_notes` 四个数组。把 JSON 整理成下面"输出格式"的结构化中文报告交给用户。

必须展示 `critical` 和 `warning` 的具体条目：

- `critical`：逐条列出所有严重问题；如果为空，明确写“未发现严重问题”。
- `warning`：逐条列出所有一般问题；如果为空，明确写“未发现一般问题”。
- 不要只输出数量，也不要静默省略问题。若问题很多，可以先按类型做简短归纳，但仍需保留完整问题列表。
- `passed` 可以摘要展示，不必逐条展开所有通过项。
- `content_notes` 作为内容质量参考展示，不要与格式错误混在一起。

### 第 3 步（可选）：用 pandoc 补充内容分析

```bash
pandoc --track-changes=all "<论文.docx路径>" -o /tmp/thesis_content.md
```

读取 Markdown 检查章节结构、文献引用等内容层面的问题。

## 检查器内部要点（修脚本 / 排查误报时参考）

- **属性继承**：`get_pPr_chain` / `get_rPr_chain` 沿"段落 → pStyle → basedOn 链 → docDefaults"做属性级回溯，所以即使对齐 / 行距 / 字号 / 字体定义在 `styles.xml` 而不是段落上，脚本也能读到。
- **页眉**：允许"本科毕业论文"或"本科毕业设计"任一变体，归一化空白后用正则匹配；跨多个 `header*.xml` 同类问题去重。
- **章节标签**：`matches_label()` 去掉所有空白并 lowercase 后比较，所以"摘 要"、"目 录"、"目　录"、"References" 等变体都能识别为对应章节。
- **TOC 嵌套**：`body.findall('.//w:p')` 递归查找，覆盖 `<w:sdt><w:sdtContent>` 包住的自动目录。
- **正文采样**：只在"第一个一级标题之后、参考文献之前"采集，跳过图表题、关键词、`[N]` 引用条目，以及代码段（`def/class/{}` 等）和非中文主导段落（中文占比 < 30%），避免代码块的 `atLeast 10.8pt` 被误判成正文行距错误。

## 格式规范速查

详细规范见 `references/format_rules.md`。

### 字号换算

| 中文字号 | pt   | XML 半磅值 |
| -------- | ---- | ---------- |
| 28号     | 28   | 56         |
| 小三     | 15   | 30         |
| 三号     | 16   | 32         |
| 四号     | 14   | 28         |
| 小四     | 12   | 24         |
| 五号     | 10.5 | 21         |

### 行距换算

| 设置         | XML line 值 | lineRule |
| ------------ | ----------- | -------- |
| 单倍行距     | 240         | auto     |
| 固定值 20 磅 | 400         | exact    |

### 段前段后

段前 / 后 2 行 ≈ 480 TWIPs（以 12pt 正文为基准）

## 注意事项

1. 脚本输出 JSON，请整理成结构化中文报告后再展示给用户。
2. 字体检查会沿继承链查找；run 上的显式覆盖会优先于样式表，这是 Word 的真实行为，不是 bug。
3. 内容质量评估只是定性参考，不要对"专业水平"等做绝对化判断。
